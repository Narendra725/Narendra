Window Functions deep dive
Jun 13, 2025 4 min read Drive · synced SQL

Window functions perform calculations across a set of rows related to the current row — unlike GROUP BY, they don't collapse rows. The result set keeps its original shape.

ROW_NUMBER, RANK, DENSE_RANK

SQL
SELECT employee_id, department, salary, ROW_NUMBER() OVER ( PARTITION BY department ORDER BY salary DESC ) AS rank_in_dept FROM employees;

Key insight: PARTITION BY resets the window per group. ORDER BY inside OVER() is independent of the outer query sort.

LEAD and LAG — row-to-row comparisons

SQL
LAG(salary, 1) OVER (ORDER BY hire_date) AS prev_salary, LEAD(salary, 1) OVER (ORDER BY hire_date) AS next_salary
